🔴
Back up everything before you start

Encryption and partitioning are irreversible. A wrong move can wipe your Windows install. Store your BitLocker recovery key somewhere safe and offline before proceeding.

Phase 01

1
Prepare Windows

STEP 1.1

Disable Fast Startup

Fast Startup puts Windows in a hybrid sleep state that locks the NTFS partitions — Linux cannot safely access them and it can cause corruption.

  1. Open Control Panel → Power Options
  2. Click "Choose what the power buttons do"
  3. Click "Change settings that are currently unavailable"
  4. Uncheck "Turn on fast startup" → Save changes
STEP 1.2

Disable Hibernation (optional but recommended)

Run as Administrator in PowerShell or CMD:

PowerShell (Administrator)
powercfg /h off
STEP 1.3

Check current partition layout

Press Win + X → Disk Management. Note the exact sizes of all partitions. You should see something like: EFI (~100MB), Windows Recovery, and your C: drive. Take a screenshot — you'll need this as a reference.

Phase 02

2
Enable BitLocker on Windows

ℹ️
Do this BEFORE shrinking the partition

BitLocker encrypts the drive in-place. It's much simpler to encrypt first, then shrink, than the other way around.

STEP 2.1

Enable BitLocker

  1. Open Control Panel → System and Security → BitLocker Drive Encryption
  2. Click "Turn on BitLocker" next to your C: drive
  3. Choose how to unlock at startup (PIN is most secure — use one you won't forget)
  4. Choose where to save the recovery key — print it or save to a USB drive, not to your Microsoft account
  5. Choose "Encrypt entire drive" (not just used space)
  6. Select "New encryption mode" (XTS-AES)
  7. Run the BitLocker check and restart

Encryption runs in the background — check the taskbar icon. Wait for it to finish completely before moving on.

⚠️
Save your BitLocker Recovery Key

It's a 48-digit code. When you change BIOS settings (like disabling Secure Boot), Windows may trigger a BitLocker recovery screen. You'll need this key to unlock. Do not skip this.

Phase 03

3
Shrink the Windows Partition

STEP 3.1

Shrink via Disk Management

  1. Open Disk Management (Win + X)
  2. Right-click your C: drive → "Shrink Volume"
  3. Enter amount to shrink: 245760 MB (≈ 240GB) — this leaves ~235-245GB for Linux depending on your current Windows size
  4. Click Shrink

You'll now see unallocated space. Leave it unallocated — the EndeavourOS installer will handle it.

💡
Can't shrink enough? Run defrag first

If Windows won't let you shrink to the size you want, open CMD as Admin and run: defrag C: /U /V then try shrinking again. Alternatively, disable the page file temporarily.

Phase 04

4
Create Bootable USB

STEP 4.1

Download EndeavourOS ISO

Go to endeavouros.com → Downloads → get the latest ISO (Gemini or newer). Use a USB drive of 8GB+.

STEP 4.2

Flash the USB

Use one of these tools:

  • Rufus (Windows, recommended) — Select ISO → GPT partition scheme → UEFI → Write in ISO Image mode
  • Ventoy — Copy the ISO onto a Ventoy-formatted drive, supports multiple ISOs
  • Balena Etcher — simplest, just point at the ISO
⚠️
Rufus mode matters

When Rufus asks, choose "Write in ISO Image mode" — NOT DD mode. Also make sure partition scheme is GPT, not MBR.

Phase 05

5
BIOS / UEFI Setup

STEP 5.1

Enter BIOS

Restart and press the BIOS key during POST. Common keys: F2, Del, F10, F12 — check your motherboard manual.

STEP 5.2

Required BIOS changes

  • ✅ Secure Boot → Disabled (you can re-enable after setup if needed, but keep off for now)
  • ✅ Boot Mode → UEFI only (disable CSM/Legacy if present)
  • ✅ Storage mode → AHCI (not RAID or Intel RST — this can prevent Linux from seeing your SSD)
  • ✅ USB Boot → Enabled
  • ✅ Boot order → USB first
⚠️
BitLocker Recovery after BIOS changes

Windows may ask for your BitLocker recovery key on the next Windows boot after changing BIOS settings. Have that 48-digit key ready.

Phase 06 — Core

6
Install EndeavourOS with LUKS Encryption

ℹ️
What we're creating

A separate unencrypted /boot partition (so GRUB can boot and then unlock LUKS), plus the encrypted root partition for everything else.

STEP 6.1

Boot into the Live Environment

With your USB plugged in, power on. The EndeavourOS live desktop will load. Connect to WiFi if needed, then click "Start the installer".

STEP 6.2

Installer: Pick Online install + desktop

When asked, choose "Online" install (gives you the most up-to-date packages). For desktop, you can choose no desktop / base if you want a fully clean Hyprland setup, or pick KDE temporarily as a fallback while you configure Hyprland.

STEP 6.3

Manual Partitioning in Calamares

When you reach the Partitions screen, choose "Manual Partitioning". This is important — do NOT choose "Erase disk" (that would wipe Windows).

Create these partitions in the unallocated space:

Partition Size Filesystem Mount Point Notes
EFI (existing) ~100–500MB FAT32 /boot/efi shared DO NOT format
Linux /boot 1024 MB ext4 /boot linux Unencrypted
Linux Root ~230–238 GB ext4 / LUKS2 Encrypted
🔴
Critical: Do NOT format the existing EFI partition

Select the existing EFI partition → set mount point to /boot/efi → leave "Format" unchecked. If you format it, Windows won't boot.

STEP 6.4

Enable LUKS on the Root Partition

When creating the root partition in Calamares:

  1. Click on the unallocated space → "Create"
  2. Set size to fill remaining space
  3. Filesystem: ext4
  4. Mount point: /
  5. Check "Encrypt this partition"
  6. Set a strong passphrase — this is what you'll type every time Linux boots
  7. Confirm passphrase
💡
Pick a good LUKS passphrase

This is typed at every boot, so balance security with usability. A 5-6 word diceware passphrase is both secure and practical. Avoid special characters that might behave differently across keyboard layouts at boot.

STEP 6.5

Complete the Installation

  1. Set your timezone, locale, and keyboard layout
  2. Create your user account (don't use root as daily driver)
  3. Bootloader: leave as GRUB (it handles dual boot automatically)
  4. Review the summary screen carefully before clicking Install
  5. Wait for installation to complete — don't interrupt it
  6. When done, click Restart Now and remove the USB when prompted
Phase 07

7
First Boot & Verification

STEP 7.1

GRUB Boot Menu

On reboot you should see the GRUB menu with at least two entries: EndeavourOS and Windows Boot Manager. Select EndeavourOS — you'll be prompted for your LUKS passphrase, then it boots into your desktop.

STEP 7.2

Verify encryption is working

Terminal
# Check LUKS is active on your root partition
lsblk -o NAME,FSTYPE,SIZE,MOUNTPOINT

# You should see something like:
# nvme0n1p4  crypto_LUKS   238G
# └─luks-xxx  ext4          238G  /

# Confirm LUKS details
sudo cryptsetup luksDump /dev/nvme0n1p4
STEP 7.3

Update the system immediately

Terminal
sudo pacman -Syu

# If it prompts about keyring:
sudo pacman -Sy archlinux-keyring endeavouros-keyring
sudo pacman -Su
Phase 08 — Performance

8
Install the CachyOS Kernel

STEP 8.1

Add the Chaotic AUR repository

The CachyOS kernel is available through Chaotic-AUR, which gives you pre-compiled AUR binaries — much faster than building from source.

Terminal
# Import Chaotic-AUR key
sudo pacman-key --recv-key 3056513887B78AEB --keyserver keyserver.ubuntu.com
sudo pacman-key --lsign-key 3056513887B78AEB

# Install the Chaotic-AUR mirrorlist and keyring
sudo pacman -U 'https://cdn-mirror.chaotic.cx/chaotic-aur/chaotic-keyring.pkg.tar.zst'
sudo pacman -U 'https://cdn-mirror.chaotic.cx/chaotic-aur/chaotic-mirrorlist.pkg.tar.zst'
STEP 8.2

Add Chaotic-AUR to pacman.conf

Terminal
sudo nano /etc/pacman.conf

# Add at the END of the file:
[chaotic-aur]
Include = /etc/pacman.d/chaotic-mirrorlist

Then sync:

Terminal
sudo pacman -Sy
STEP 8.3

Install the CachyOS kernel (BORE scheduler)

Terminal
# Install kernel + headers (headers needed for Docker, dkms modules)
sudo pacman -S linux-cachyos linux-cachyos-headers

# Regenerate GRUB to include the new kernel
sudo grub-mkconfig -o /boot/grub/grub.cfg

Reboot. In GRUB, choose Advanced Options → select the CachyOS kernel. Once you verify it boots correctly, you can set it as default.

STEP 8.4

Set CachyOS kernel as default (optional)

/etc/default/grub
# Find the exact menu entry name first:
sudo grub-mkconfig | grep -i cachy

# Then set in /etc/default/grub:
GRUB_DEFAULT="Advanced options for EndeavourOS>EndeavourOS, with linux-cachyos"

sudo grub-mkconfig -o /boot/grub/grub.cfg
Phase 09

9
Dev Environment Setup

STEP 9.1

Install Hyprland and essentials

Terminal
sudo pacman -S hyprland waybar \
  kitty wofi \
  xdg-desktop-portal-hyprland \
  pipewire pipewire-pulse wireplumber \
  polkit-kde-agent qt5-wayland qt6-wayland \
  thunar gvfs dunst \
  swww hyprpaper \
  cliphist wl-clipboard
STEP 9.2

Install Docker

Terminal
sudo pacman -S docker docker-compose

# Enable and start Docker
sudo systemctl enable --now docker

# Add yourself to the docker group (no sudo needed)
sudo usermod -aG docker $USER

# Log out and back in, then verify:
docker run hello-world
STEP 9.3

Install dev tools from AUR

First, install an AUR helper (yay or paru):

Terminal
# Install paru (Rust-based, fast)
sudo pacman -S --needed base-devel git
git clone https://aur.archlinux.org/paru.git
cd paru && makepkg -si

# Now install common dev tools:
paru -S visual-studio-code-bin postman-bin \
  nvm nodejs npm \
  python python-pip \
  lazygit \
  github-cli \
  zsh starship \
  btop
STEP 9.4

Set up zsh + starship prompt

Terminal
# Set zsh as default shell
chsh -s $(which zsh)

# Add starship init to ~/.zshrc
echo 'eval "$(starship init zsh)"' >> ~/.zshrc

# Install zsh-autosuggestions and syntax highlighting
paru -S zsh-autosuggestions zsh-syntax-highlighting

# Add to ~/.zshrc:
echo 'source /usr/share/zsh/plugins/zsh-autosuggestions/zsh-autosuggestions.zsh' >> ~/.zshrc
echo 'source /usr/share/zsh/plugins/zsh-syntax-highlighting/zsh-syntax-highlighting.zsh' >> ~/.zshrc
STEP 9.5

Performance tuning for dev workloads

Terminal
# Increase inotify watchers (needed for file-heavy dev servers)
echo 'fs.inotify.max_user_watches=524288' | sudo tee /etc/sysctl.d/99-dev.conf
sudo sysctl --system

# Set swappiness low (32GB RAM — you barely need swap)
echo 'vm.swappiness=10' | sudo tee -a /etc/sysctl.d/99-dev.conf

# Enable trim for your SSD
sudo systemctl enable fstrim.timer

Final checklist before you start
  • Full backup of Windows data
  • BitLocker recovery key saved offline
  • LUKS passphrase decided (memorable but strong)
  • 8GB+ USB drive available
  • Stable power supply (don't do this on battery)
  • ~2–3 hours set aside for the full process
